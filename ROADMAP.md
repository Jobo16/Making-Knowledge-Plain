# Roadmap

## Current stage

The project is currently in a methodology-building phase.

It already has a solid first-pass foundation:
- principles
- rewriting rules
- target language spec
- quality checklist
- benchmark references
- a deeper research playbook

The next step is to turn that foundation into a reusable working system.

## Near-term priorities

### 1. Build a case library

Create a structured set of examples that show:
- original text
- faithful rewrite
- explanation layer
- critique layer
- review notes

This is the fastest way to make the project concrete.

### 2. Create reusable templates

Add templates for:
- rewrite tasks
- review tasks
- research logging
- domain-specific handling

### 3. Define the problem taxonomy more formally

The current problem ontology is strong in spirit. The next step is to make it easier to apply consistently across cases.

### 4. Establish a review workflow

Every output should be easy to inspect for:
- fidelity
- clarity
- evidence visibility
- concision
- actionability

### 5. Add domain tracks

Some domains need stricter handling than others.

Suggested early tracks:
- philosophy / theory
- policy / public communication
- technology / AI writing
- business / management writing

## Medium-term priorities

### 1. Benchmark set

Build a benchmark collection of difficult passages and expected outputs.

### 2. Review protocol

Standardize how a rewrite gets approved, revised, or rejected.

### 3. Terminology handling guide

Create a repeatable method for deciding when to:
- keep a term
- define a term
- replace a term
- split a term into observable actions or relations

### 4. Publishing structure

Make the repository easier for newcomers to navigate by separating:
- project philosophy
- operating rules
- examples
- research
- templates

## Long-term direction

The project can become more than a set of notes.

It could become:
- a public methodology for plain-knowledge rewriting
- a benchmark and review framework
- a case library of before/after examples
- a practical toolkit for writers, researchers, and AI systems

## Decision rule for what to do next

If unsure what to work on next, prefer this order:

1. add examples
2. improve reviewability
3. improve repeatability
4. expand research
5. expand scope

That order keeps the project grounded.
