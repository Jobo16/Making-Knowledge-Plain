# 项目起点说明

这个项目起源于一个很直接的观察：

很多作品并不是在帮助人理解，而是在用沉重、模糊、陌生的语言制造理解门槛。

一些本来可以被说清楚的问题，被包装成只有特定圈层才能进入的话语。结果不是知识更深了，而是知识更远了。

## 初衷

这个项目的初衷，不是把内容“说浅”，而是让内容重新回到人能直接进入的语言里。

它相信：

- 很多思想本身并不该被表达门槛垄断
- 复杂不等于模糊
- 清楚不等于浅薄
- 降低门槛，不等于降低内容

## 历史灵感

这个项目带有一种“知识白话化”的方向感。

当知识长期停留在封闭表达中时，真正被挡住的不是阅读技巧，而是理解权。

因此，这个项目不是单纯的语言润色项目，而是一个关于知识传播、表达透明和理解进入权的项目。

## 要解决的核心问题

### 1. 复述会不会偏离原意？

会有风险，所以项目必须建立严格约束。

复述不是借题发挥，而是在原文边界内重建可理解形式。

### 2. 转化规则是什么？

必须有明确规则，不能靠感觉：

- 保留问题
- 保留逻辑
- 保留边界
- 不擅加新意
- 不用新黑话替换旧黑话

### 3. 目标语言应该是什么样？

不是越白越好，也不是越口语越好。

目标语言应该是：

- 普通但准确
- 清楚但不空
- 简洁但不失真
- 可进入但不口水化

## 方法入口

本项目目前确定的四个入口是：

1. 从现象出发
2. 从问题出发
3. 从经验出发
4. 不默认作者比读者高明

## 工作标准

这个项目追求的不是“简单化”，而是：

**低阻力，高保真。**

也就是说：

- 让更多人能进入
- 尽量不扭曲原来的东西
- 去掉不必要的进入障碍
- 保留真正复杂的部分

## 当前阶段

当前版本仍处于方法论搭建阶段。

后续应逐步补充：

- 示例拆解
- 各类文本模板
- 术语处理案例
- 忠实复述 / 辅助解释 / 批判性拆解 的边界示例

## 一句话总结

不是把复杂的东西说浅，而是把遮住理解的东西拿掉，让真正复杂的部分以人能进入的方式出现。
