# 目标语言规范 v1

本项目的目标语言，不是儿童化语言，不是闲聊口语，不是鸡汤，也不是稀释版摘要。

它应该是一种：

**普通但准确，清楚但不空，简洁但不失真。**

## 一、目标

让一个普通但认真读的人，不借助术语，也能直接接触原文真正想表达的问题、逻辑和力度。

## 二、四个标准

### 1. 读者第一次读就能建立画面或问题感

表达应尽量让读者知道：

- 这是在说什么现实
- 这是在解释什么问题
- 这和人的生活经验有什么关系

### 2. 抽象名词尽量少，动作和关系尽量多

一个句子里如果堆满抽象名词，理解就容易打滑。

优先使用：

- 动作
- 关系
- 处境
- 冲突
- 感受
- 结果

### 3. 信息密度要高，但阅读阻力要低

去术语化不等于变啰嗦。

好的目标语言应该：

- 词更熟悉
- 句更清楚
- 信息不缩水
- 废话不增殖

### 4. 语言看起来简单，但能反推出原文骨架

好的复述不能只求顺口。

读者读完之后，应该仍能感受到：

- 原文在处理什么问题
- 原文是怎么组织逻辑的
- 原文的判断力度在哪里

## 三、推荐语言特征

- 句子短，但不碎
- 词汇常用，但不俗烂
- 结构直接，但不粗糙
- 表达具体，但不缩窄原意
- 逻辑清楚，但不生硬

## 四、应避免的语言问题

### 1. 口水化

错误例子：

“其实简单来说呢，人有时候吧，可能会因为一些外界原因，然后慢慢地……”

问题：拖沓、稀释、无信息。

### 2. 鸡汤化

错误倾向：

把原文中的冲突、控制、失衡、压迫、异化、断裂等尖锐内容，翻成温和安抚的话。

### 3. 缩水化

错误倾向：

为了让句子短，直接把关键条件、逻辑关系、区分和限定删掉。

### 4. 聊天化

错误倾向：

写得像社交媒体安抚发言，而不是可检验的复述。

## 五、建议写法

### 写法一：先说问题，再说判断

先让读者知道它在解释什么，再给出复述后的理解。

### 写法二：先说现象，再抽出结构

先落地，再上提。

### 写法三：先给场景，再给概括

让读者先进入，再总结。

## 六、一句话标准

目标语言应该做到：

**读起来像人话，拆开看有骨架。**
